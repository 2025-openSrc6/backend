## 1. 블록체인이 뭐야?

### 한 줄 설명

> **전 세계 컴퓨터가 공유하는 변조 불가능한 데이터베이스**

### 특징

| 특징       | 설명                    | 비유                        |
| ---------- | ----------------------- | --------------------------- |
| **분산**   | 한 곳에 저장 안 함      | 구글 드라이브 ❌, 토렌트 ✅ |
| **불변**   | 한번 기록하면 수정 불가 | 연필 ❌, 볼펜 ✅            |
| **투명**   | 누구나 기록 볼 수 있음  | 비밀 금고 ❌, 유리 금고 ✅  |
| **탈중앙** | 관리자 없음             | 은행 ❌, 비트코인 ✅        |

### 왜 쓰는가?

```
기존 시스템:
- 서버 관리자가 데이터 조작 가능
- 서버 다운되면 서비스 중단
- "내 돈이 여기 있다"는 회사 약속에 의존

블록체인:
- 누구도 데이터 조작 불가능
- 분산 저장이라 다운 없음
- 코드(스마트 컨트랙트)가 보장 → "믿지 마, 검증해"
```

## 2. Sui의 특징

### Sui는 뭐가 다른가?

| 특징            | 이더리움      | Sui             |
| --------------- | ------------- | --------------- |
| **속도**        | 느림 (15 TPS) | 빠름 (수만 TPS) |
| **수수료**      | 비쌈          | 저렴            |
| **데이터 모델** | Account 기반  | Object 기반     |
| **언어**        | Solidity      | Move            |

### Object 기반의 장점

```
이더리움 (Account 기반):
- 모든 데이터가 하나의 계정에 묶임
- 동시 처리 어려움

Sui (Object 기반):
- 각 데이터가 독립적인 Object
- 관련 없는 Object들 동시 처리 가능 → 빠름!
```

## 3. 지갑 (Wallet)

### 지갑 = 은행 계좌

```
┌─────────────────────────────────────────┐
│  Sui 지갑                                │
│                                         │
│  주소 (Address):                        │
│  0xb092f93ec3605a42c99d421ccbec14a33... │
│  → 계좌번호. 공개해도 됨                 │
│                                         │
│  비밀키 (Secret Key):                   │
│  suiprivkey1qpqg5vh0uxq3742mlf8vv3...   │
│  → 비밀번호. 절대 공개 금지!            │
│                                         │
│  보유 자산:                              │
│  - 1.96 SUI                             │
│  - AdminCap (Object)                    │
│  - TreasuryCap (Object)                 │
└─────────────────────────────────────────┘
```

### 핵심 규칙

```
✅ 주소는 공개해도 됨 → 누가 내 계좌번호 알아도 돈 못 빼감
❌ 비밀키는 절대 공개 금지 → 비밀번호 알면 돈 다 털림
```

### 지갑 생성 방법

1. **CLI로 생성** (개발자용)

   ```bash
   sui client new-address ed25519
   ```

2. **브라우저 지갑** (일반 사용자용)
   - Slush Wallet: https://slush.app
   - Sui Wallet (공식)

## 4. Object 모델

### Object가 뭐야?

Sui에서 **모든 데이터는 Object(객체)**로 저장됨.

```
┌─────────────────────────────────────────────────────────────┐
│                    Sui 블록체인                              │
│                                                             │
│   ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐       │
│   │   SUI   │  │AdminCap │  │  Pool   │  │   Bet   │       │
│   │  Coin   │  │         │  │         │  │         │       │
│   │         │  │         │  │         │  │         │       │
│   │ Owner:  │  │ Owner:  │  │ Owner:  │  │ Owner:  │       │
│   │ 0xAAA   │  │ 0xAAA   │  │ Shared  │  │ 0xBBB   │       │
│   └─────────┘  └─────────┘  └─────────┘  └─────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Object 소유 유형

| 유형          | 설명                   | 예시                    |
| ------------- | ---------------------- | ----------------------- |
| **Owned**     | 특정 주소가 소유       | Coin, AdminCap, Bet     |
| **Shared**    | 누구나 접근 가능       | BettingPool             |
| **Immutable** | 변경 불가, 누구나 조회 | Package (컨트랙트 코드) |

### 왜 중요해?

```
Owned Object:
- 소유자만 사용 가능
- 병렬 처리에 유리

Shared Object:
- 여러 사람이 동시 접근 필요할 때
- 베팅 풀처럼 모두가 참여해야 하는 경우
```

## 5. 트랜잭션

### 트랜잭션이 뭐야?

> **블록체인에 변화를 주는 요청**

```
예시:
- 코인 전송: "0xAAA에서 0xBBB로 100 SUI 보내줘"
- 함수 호출: "betting::place_bet() 실행해줘"
- 배포: "이 코드를 블록체인에 저장해줘"
```

### 트랜잭션 생명주기

```
1. 생성 (Build)
   어떤 작업을 할지 정의
        │
        ▼
2. 서명 (Sign)
   비밀키로 "내가 승인함" 증명
        │
        ▼
3. 제출 (Submit)
   Sui 네트워크에 전송
        │
        ▼
4. 실행 (Execute)
   네트워크가 검증 후 실행
        │
        ▼
5. 확정 (Finalize)
   결과가 영구 저장됨
```

### 트랜잭션 결과

```json
{
  "digest": "4AC28oaeNBU9BJjpgS52BkcaS8ZpWG5tRor9GLi1EQqN",
  "status": "success",
  "effects": {
    "created": [...],
    "mutated": [...],
    "deleted": [...]
  }
}
```

- **digest**: 트랜잭션 고유 ID (영수증 번호)
- **status**: 성공/실패
- **effects**: 어떤 Object가 생성/수정/삭제됐는지

## 6. Gas (가스비)

### Gas가 뭐야?

> **블록체인 사용료** (전기세 같은 것)

### 왜 필요해?

```
문제: 무한 루프 공격
for (;;) { } // 무한 루프

해결: 연산마다 Gas 소모
- 연산할 때마다 SUI 차감
- SUI 다 쓰면 자동 중단
- 공격자는 돈만 날림
```

### Gas 관련 용어

| 용어           | 설명                      |
| -------------- | ------------------------- |
| **Gas**        | 연산량 단위               |
| **Gas Price**  | Gas 1단위당 가격 (MIST)   |
| **Gas Budget** | 최대 이만큼 쓰겠다고 선언 |
| **Gas Cost**   | 실제 사용량               |

### 예시

```bash
sui client publish --gas-budget 200000000
#                              ↑
#                   200,000,000 MIST = 0.2 SUI

# 실제 결과:
# Gas Cost: 40,035,880 MIST = 0.04 SUI
# 나머지 0.16 SUI는 돌려받음
```

### MIST vs SUI

```
1 SUI = 1,000,000,000 MIST (10^9)
= 10억 MIST

MIST = SUI의 최소 단위 (원화의 '원' 같은 것)
```

## 7. Move 언어

### Move가 뭐야?

Sui 스마트 컨트랙트를 작성하는 프로그래밍 언어.

### 특징

| 특징          | 설명                                      |
| ------------- | ----------------------------------------- |
| **안전성**    | 자산 관련 버그 방지 (복제, 삭제 등)       |
| **자원 중심** | 모든 것이 "자원" (복사 불가, 이동만 가능) |
| **정적 타입** | 컴파일 시 오류 발견                       |

### 기본 구조

```move
module deltax::betting {
    // Struct (데이터 구조)
    struct BettingPool has key {
        id: UID,
        total_pool: u64,
    }

    // Function (함수)
    public fun create_pool(ctx: &mut TxContext): ID {
        // ...
    }
}
```

### 중요 개념: Abilities

```move
struct MyStruct has key, store, copy, drop { }
```

| Ability   | 설명                       |
| --------- | -------------------------- |
| **key**   | Object로 저장 가능         |
| **store** | 다른 Object 안에 저장 가능 |
| **copy**  | 복사 가능                  |
| **drop**  | 자동 삭제 가능             |

```
돈(Coin)은 copy 없음 → 복제 불가능 → 안전!
```

## 8. CLI vs 브라우저 지갑

### 완전 다른 것!

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   CLI 지갑 (터미널)              브라우저 지갑 (Slush 등)   │
│   ──────────────────            ───────────────────────     │
│                                                             │
│   위치: ~/.sui/                 위치: 브라우저 확장 프로그램│
│                                                             │
│   주소: 0xAAA...                주소: 0xBBB...              │
│   (다른 주소!)                  (다른 주소!)                │
│                                                             │
│   용도: 개발, 배포              용도: 일반 사용, 테스트     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 같은 지갑 쓰려면?

**CLI → 브라우저:**

```bash
# CLI에서 비밀키 추출
sui keytool export --key-identity main

# 브라우저 지갑에서 Import
# 설정 → Import Wallet → Private Key 붙여넣기
```

**브라우저 → CLI:**

```bash
# 브라우저 지갑에서 비밀키 Export
# 설정 → Export Private Key

# CLI에서 Import
sui keytool import <비밀키> ed25519
```
